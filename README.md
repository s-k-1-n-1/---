# Weather & System Monitor

## Описание проекта

Данный проект представляет собой веб-приложение для получения и отображения данных о погоде, а также взаимодействия с локальным сервером.

Приложение построено по клиент-серверной архитектуре и демонстрирует интеграцию frontend и backend.

---

## Цель проекта

Разработка программного комплекса, который:

* получает данные из внешнего API (погода)
* обрабатывает их на сервере
* передаёт на клиент
* отображает в удобном интерфейсе

---

## Архитектура системы

Приложение работает по следующей схеме:

Frontend → Backend → OpenWeather API → Backend → Frontend

Описание:

* Клиент отправляет запрос на сервер
* Сервер принимает запрос
* Сервер обращается к внешнему API
* Полученные данные возвращаются в формате JSON
* Интерфейс обновляется без перезагрузки страницы

## Функциональные возможности

### Модуль погоды

* Ввод названия города
* Получение температуры
* Получение описания погодных условий

### Модуль интеграции

* Проверка соединения с сервером
* Получение тестовых данных с endpoint `/api/data`

### Динамическое обновление

* Обновление интерфейса без перезагрузки страницы
* Асинхронные запросы (fetch)

### Обработка ошибок

* Сообщение при неверном городе
* Обработка ошибок сервера

---

## Безопасность

API-ключ хранится на стороне сервера и не передаётся клиенту, что предотвращает его утечку.

## Установка и запуск

### 1. Клонирование репозитория

git clone https://github.com/USERNAME/REPOSITORY

---

### 2. Запуск backend

Перейти в папку:
cd backend

Установить зависимости:
npm install

Запустить сервер:
node server.js

После запуска появится:
Сервер работает на http://localhost:3000

---

### 3. Запуск frontend

Открыть файл:
frontend/index.html

в браузере.

---

## API endpoints

### 1. Тестовый endpoint

GET /api/data

Возвращает:
{
"data": {
"title": "Интеграция успешно завершена!",

}

---

### 2. Погода

Пример:
http://localhost:3000/api/weather?city=London
